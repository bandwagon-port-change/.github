## 准备工作：在开始之前确认两件事

### 确认一：是 IP 被封，还是端口被封？

这是两个不同的问题，解决方法也完全不同。

先搞清楚情况：

1. 打开任意一个端口检测网站，比如 `port.ping.pe`
2. 输入你的搬瓦工 IP 和当前 SSH 端口（在 KiwiVM 后台首页可以看到）
3. 看结果：
   - 如果"境外节点"都显示 **成功（green）**、只有大陆节点显示 **失败（red）**，说明端口被封了，本文就是为你准备的
   - 如果**全部节点**都连不上，那很可能是 IP 被封，需要另行付费换 IP（目前搬瓦工换 IP 收费约 8 美元）

端口被封 ≠ IP 被封。确认是端口问题之后，我们继续。

### 确认二：找到你的 KiwiVM 后台登录方式

KiwiVM 是搬瓦工自家开发的控制面板，你的 VPS 信息、重装系统、重启、Web 终端都在这里。

登录路径：

KiwiVM 里有一个功能叫 **Root shell – interactive**，它就是一个网页版的终端，可以直接操作服务器。端口被封之后 SSH 连不上，但 KiwiVM 的 Web 终端不受影响，正是我们这次操作的入口。

## 第一步：进入 KiwiVM 的 Web 终端

1. 登录 KiwiVM 后台（方法见上）
2. 在左侧菜单中找到 **Root shell – interactive**
3. 点击 **Launch** 按钮
4. 弹出的窗口里，输入用户名 `root`，回车
5. 输入你的 root 密码（**注意：输密码时不会显示任何字符，这是正常的**），回车

成功进入后，你会看到命令提示符 `[root@xxx ~]#`，现在你已经有了 root 权限。

> **小提示**：如果你还能正常 SSH 连接服务器，也可以直接用 XShell、Tabby、PuTTY 等工具连上来操作，效果是一样的。KiwiVM Web 终端是 SSH 连不上时的备用方案。

## 第二步：选择一个新端口

在动手之前，先想好要换成哪个端口。

理想的新端口满足这几点：

- **不是常见服务的默认端口**：避免 80（HTTP）、443（HTTPS）、3306（MySQL）、3389（远程桌面）等
- **范围在 1024–65535 之间**：1–1023 是系统保留端口
- **容易记住**：比如 29200、38976、51288 这类四五位数的随机数

以下演示统一用 **29200** 作为新端口示例，你换成自己选的数字即可。

---

## 第三步：防火墙放行新端口

**先放行，再改配置**——这个顺序非常重要。如果先改完配置、重启 SSH 服务，但防火墙没放行，会直接把自己锁在外面。

在终端里执行以下命令（选对应你系统的那组）：

**CentOS 7 / 使用 firewalld 的系统：**

bash
firewall-cmd --zone=public --add-port=29200/tcp --permanent
firewall-cmd --reload


**Ubuntu / Debian / 使用 iptables 的系统：**

bash
iptables -A INPUT -p tcp --dport 29200 -j ACCEPT


执行完之后，可以用下面这条命令确认端口是否已经被放行（firewalld）：

bash
firewall-cmd --list-ports


看到 `29200/tcp` 出现在列表里，说明放行成功。

---

## 第四步：修改 SSH 配置文件

SSH 的端口配置写在 `/etc/ssh/sshd_config` 这个文件里。我们来打开它。

**方法一：用 sed 命令一行搞定（推荐）**

bash
sed -i 's|^#Port 22|Port 29200|' /etc/ssh/sshd_config


这条命令会自动找到 `#Port 22` 这一行，把它改成 `Port 29200`。

**方法二：用 vim 手动编辑**

bash
vim /etc/ssh/sshd_config


如果提示 vim 不存在，先安装：

bash
# Ubuntu/Debian
apt install -y vim

# CentOS
yum -y install vim


打开文件后，按 **/** 搜索 `Port`，找到 `#Port 22` 这一行，按 **i** 进入编辑模式，把它改为：


Port 29200


注意：**把前面的 `#` 号删掉**，否则这行是注释，不生效。

修改完后，按 **Esc** 退出编辑模式，输入 `:wq` 回车保存。

**验证改动是否生效：**

bash
grep "^Port" /etc/ssh/sshd_config


如果输出 `Port 29200`，说明修改已经写入。

---

## 第五步：重启 SSH 服务

配置改好了，需要重启 SSH 服务让新配置生效。

**CentOS 6 / 旧版系统：**

bash
/etc/init.d/sshd restart


**CentOS 7 / Ubuntu / Debian（systemd 系统）：**

bash
systemctl restart sshd


或者：

bash
service ssh restart


重启完成后，**先不要关闭当前的 KiwiVM Web 终端窗口**。

---

## 第六步：验证新端口能否连接

在 KiwiVM Web 终端还开着的情况下，**另外开一个新的 SSH 连接**，用新端口试着连一下：


ssh root@你的IP -p 29200


如果能成功进入，说明换端口完成。这时候可以安全关闭 KiwiVM 的 Web 终端了。

如果新端口还是连不上，回到 KiwiVM 终端排查：

bash
# 查看 SSH 服务状态
systemctl status sshd

# 查看当前监听的端口
ss -tlnp | grep ssh


确认 SSH 服务在新端口上正常监听，再检查防火墙规则有没有遗漏。

---

## 第七步：更新本地 SSH 客户端配置

换好端口之后，别忘了更新本地工具里保存的连接信息。

- **XShell / Tabby / PuTTY**：打开连接配置，把端口号从原来的改成新的 29200（或你设定的端口）
- **macOS / Linux 终端**：如果用 `~/.ssh/config` 文件管理连接，把对应主机的 `Port` 字段更新一下

同时，把新端口号记录在一个安全的地方。KiwiVM 后台首页显示的 SSH Port 是初始端口，手动改了之后那里不会自动同步，需要自己记。

## 常见错误与排查

**Q：执行完重启命令之后，用新端口连不上怎么办？**

按顺序检查：
1. 防火墙规则是否正确放行了新端口（回看第三步）
2. SSH 配置文件里 Port 前面的 `#` 是否已经删除
3. SSH 服务是否正常运行（用 `systemctl status sshd` 查看）
4. 新端口是否已经在监听（用 `ss -tlnp | grep ssh` 确认）

**Q：操作中把自己锁在外面了怎么办？**

回到 KiwiVM 后台，用 **Root shell – interactive** 进入 Web 终端，从里面修复配置。这是搬瓦工的一大优势：即使 SSH 完全连不上，KiwiVM Web 终端依然可以进入服务器。

**Q：多久需要换一次端口？**

没有固定周期。被封了就换，平时不用主动折腾。如果想长期防患于未然，选一个偏门的随机端口就好，不需要频繁更换。

**Q：除了换端口，还有什么安全加固措施？**

换端口只是第一步，后续可以考虑：
- 禁用 root 密码登录，改用 SSH 密钥认证
- 安装 fail2ban 防止暴力破解
- 定期检查 `last` 命令查看异常登录记录
